Routes

This section outlines the routes available in different parts of the application.

Dashboard Routes

These routes handle the dashboard functionality.

app.services.dashboard_routes.get_complained_orders()

Retrieves the orders that have complaints for a specific year, month, store, and shipping company.

Parameters:

  • year (int) – The year for which complained orders are to be retrieved.

  • month (int, optional) – The month for which complained orders are to be retrieved.

  • store (str, optional) – The store for which complained orders are to be retrieved.

  • shipping_company (str, optional) – The shipping company for which complained orders are to be retrieved.

Returns:

A JSON object containing the complained orders, or an error message if the parameters are invalid.

Return type:

Response

Data Unit Routes

These routes deal with operations related to data units.

app.services.data_unit_routes.add_package()

Retrieves all available data units.

This endpoint fetches all the data units stored in the system. If no data units are available, it returns a message indicating that no data units are available. If data units are found, they are returned as part of the response.

Returns:

A JSON object containing the data units and an optional message, or an empty list if no data units are found.

Return type:

Response

app.services.process_mining_routes.proccess_model_start()

Starts the process model generation based on the provided parameters.

This endpoint initiates the creation of a process model based on the specified graph type (e.g., performance, frequency, Petri net) and other parameters like complexity and filters. It runs the model generation in a separate process to avoid blocking.

Returns:

A JSON object indicating the status of the request and the filename.

Return type:

Response

app.services.process_mining_routes.proccess_model_verify()

Verifies the status of the process model generation.

This endpoint checks if the SVG file representing the process model is generated. If it exists, it returns the file. If the file is still being generated, it responds with a message indicating that the SVG is being generated.

Returns:

The generated SVG file or a status message indicating the current process state.

Return type:

Response

Statistics Routes

These routes provide access to statistical analysis features.

app.services.statistics_routes.evolution_data()

Retrieve evolution data for deliveries based on the provided filters.

This endpoint fetches evolution data about deliveries over time based on the provided filters like shipping company, store, product ID, order ID, and more.

Parameters:

  • transportadora (str) – The shipping company for filtering orders.

  • loja (str) – The store for filtering orders.

  • product_id (str) – The product ID for filtering orders.

  • order_id (str) – The order ID for filtering orders.

  • product_type (str) – The type of product for filtering orders.

  • distribution_center (str) – The distribution center for filtering orders.

  • transportation_method (str) – The transportation method for filtering orders.

  • delivery_type (str) – The delivery type for filtering orders.

  • classification (str) – The classification for filtering orders.

  • month (str) – The month for filtering evolution data.

  • year (str) – The year for filtering evolution data.

Returns:

A JSON response containing the evolution data. status_code: HTTP status code 200.

Return type:

jsonify

app.services.statistics_routes.get_complained_orders_by_year_month()

Retrieve complained orders by year and month.

This endpoint fetches statistics of complained orders based on the provided year, month, store, shipping company, product ID, and package type ID.

Parameters:

  • year (int) – The year for the order statistics.

  • month (int) – The month for the order statistics.

  • store (str) – The store for filtering orders.

  • shipping_company (str) – The shipping company for filtering orders.

  • product_id_substring (str) – The substring of product ID for filtering orders.

  • package_type_id_substring (str) – The substring of package type ID for filtering orders.

Returns:

A JSON response containing the list of complained orders. status_code: HTTP status code 200 or 400 if there are validation errors.

Return type:

jsonify

app.services.statistics_routes.get_complained_packages_by_last_days_or_months()

Retrieve complained orders by the last specified days or months.

This endpoint fetches the statistics for complained orders based on the time frame provided in days or months. The time frame can be 7, 15, or 30 days or 3, 6, or 12 months.

Parameters:

  • days (int) – The number of days for the order statistics.

  • months (int) – The number of months for the order statistics.

  • year (int) – The year for the order statistics.

  • month (int) – The month for the order statistics.

  • store (str) – The store for filtering orders.

  • shipping_company (str) – The shipping company for filtering orders.

  • product_id_substring (str) – The substring of product ID for filtering orders.

  • package_type_id_substring (str) – The substring of package type ID for filtering orders.

Returns:

A JSON response containing the list of complained orders. status_code: HTTP status code 200 or 400 if there are validation errors.

Return type:

jsonify
